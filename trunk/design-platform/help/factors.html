Factors

This tab is used to gather information about factors of the experiment.

Factor tag: Touchstone can create three basic experiment designs: between-subjects, within-subjects and mixed. Touchstone does this by specifying which factors are between-subjects and which are within-subjects.

The values of a between-subjects factor are randomly assigned to subjects (or groups of subjects). This means that each subject (or group) experienced a different value of this factor. On the opposite, the values of a within-subjects factor are all assigned to each subject. A pure between-subjects (resp. within-subjects) involves only between-subjects (resp. within-subjects) factors. A mixed design involves both between-subjects and within-subjects factors.

Factor name: The name of a factor is made of two elements: a unique id of a few letters that will basically be used when reporting experiment design and results (text and graphs) and a long name that is more explicit.

Factor type: The type is either character or number. It is independent from the "statistical type" (nominal, ordinal, etc.), the type here is used to generate the Java code for factors. Factors of type character typically inherit from class CharacterFactor while factors of type number inherit from class NumericalFactor. Factors of type character have basically string values but note that you will be able to use more complex Java object values with the run platform. [TODO explain this somewhere]

Factor values: As factors, values have an id and an (optional) long name. Be sure to be consistent between the factor type and the factor values (e.g., you can not add a string value to a factor of type character). A value can be deleted by selecting it in the list and pressing the backspace key.
